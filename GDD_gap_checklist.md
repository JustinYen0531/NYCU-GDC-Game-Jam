# GDD 內容差距檢查清單

> 用途：快速衡量「目前專案內容」距離完整 GDD 還差多少。
>
> 使用方式：
> - `[x]` 已完成
> - `[-]` 已有部分內容，但還不完整
> - `[ ]` 尚未開始
>
> 建議：
> - 每一大項先只判斷有沒有
> - 第二輪再補「缺什麼」

## 1. 專案核心

- [x] 遊戲名稱已確定
- [x] 一句話介紹遊戲已寫好
- [x] 遊戲類型已明確
- [-] 目標平台已明確
- [ ] 目標玩家已明確
- [x] 遊戲核心賣點已整理
- [x] Game Jam 題目和遊戲的連結已說清楚

### 還缺什麼

- [x] 能在 30 秒內講完這款遊戲在玩什麼
- [x] 能清楚說出「為什麼這款遊戲有趣」
- [-] 能清楚說出與其他作品的差異
