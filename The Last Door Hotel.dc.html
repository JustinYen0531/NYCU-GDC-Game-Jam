專案設計文件 // PROJECT GDD

[00 設計哲學]

* 核心情緒：掌控與恐慌的博弈。
* 設計意圖：玩家扮演監視器後的「實體旅館老闆」，必須親自在門與門之間奔跑。綠色終端美學與實體開關門機制的結合，創造出一種幽閉恐懼且高度緊張的氛圍。

[01 電梯簡報]

玩家在復古綠色 CRT 監視終端界面中，控制必須親自跑去開關與鎖門的旅館老闆，在多種門機制、緩慢傳送的鏡子通道、以及會通報稽查員的大膽旅客限制下，設法隱瞞女兒的鬼魂存在，並防止旅館陷入靈異混亂。

[02 核心玩法]

1. 實體走位與門鎖互動：操控老闆奔跑，靠近門以進行手動鎖定、解鎖、開啟或關閉。
2. 監視與路徑預判：在監視畫面中觀察旅客和鬼魂，預判他們何時相遇。
3. 威脅管理與阻斷：防範膽小旅客變鬼、阻截大膽旅客通報、與稽查員周旋。

[03 核心循環] 
[觀察監視器] -> [跑向特定的門] -> [門邊鎖門/開門] -> [隔離/重導向鬼魂] -> [度過夜晚]
[04 遊戲賣點]

* 終端機美學：綠色磷光螢幕特效，帶有掃描線與鏡頭畸變。
* 實體開關門：玩家不能隔空點擊，必須跑去門邊互動。
* 連鎖混亂：受驚旅客變鬼，鬼魂穿牆，大膽旅客引來強大稽查員。

[08 地圖與門控制]

* 標準門：玩家手動開關與上鎖。
* 自動門：每隔幾秒會自動開閉，無法鎖死。
* 雙向連動門：成對連動，關一開一。
* 鏡子傳送門：鬼魂傳送通道，傳送時有長引導光芒。

[09 角色設定]

* 普通旅客 (Basic)：受驚嚇度滿 100% 會轉化為新的遊蕩鬼魂。
* 大膽旅客 (Brave)：不會變鬼，見鬼後逃出旅館通報，使稽查員提前抵達。
* 稽查員 (Inspector)：高速巡邏，具備衝刺位移。撞見鬼魂或靈異現象即判定玩家落敗。
